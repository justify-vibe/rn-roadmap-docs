# 路线图介绍

## 这份路线图适合谁？

- 完全零基础，想做一个手机 App
- 有一点点 JavaScript 或网页经验
- 想用一套代码同时覆盖 iOS 和 Android

## 整体学习路径

```
JavaScript 基础
      ↓
React 核心概念
      ↓
React Native 入门（Expo）
      ↓
进阶功能（网络/存储/动画）
      ↓
完整项目实战
      ↓
打包 & 上架发布
```

## 时间预期

| 阶段 | 内容 | 建议时长 |
|------|------|----------|
| 阶段一 | 基础知识储备 | 2～4 周 |
| 阶段二 | React Native 入门 | 3～4 周 |
| 阶段三 | 进阶功能开发 | 4～6 周 |
| 阶段四 | 完整项目开发 | 4～8 周 |
| 阶段五 | 打包 & 发布上架 | 1～2 周 |

> **总计约 3～6 个月**，每天投入 1～2 小时。进度取决于你的基础和每天的学习时间。

## 为什么选 React Native？

**跨平台**：一套 JavaScript 代码，同时运行在 iOS 和 Android。

**生态成熟**：Meta 维护，Meta、Microsoft、Shopify 等大厂在生产环境使用。

**接近原生**：不是 WebView 套壳，渲染的是真实原生组件，性能有保障。

**招聘需求大**：国内外对 RN 开发者的需求量持续稳定。

## React Native vs Flutter

| 对比项 | React Native | Flutter |
|--------|-------------|---------|
| 语言 | JavaScript / TypeScript | Dart |
| 上手难度 | 有 JS 基础较易 | 需学 Dart |
| 生态 | npm 生态庞大 | 相对较小 |
| 性能 | 接近原生 | 接近原生 |
| 推荐场景 | 前端背景转型 | 从零学习 |

如果你已经会 JavaScript，RN 是更平滑的选择。
